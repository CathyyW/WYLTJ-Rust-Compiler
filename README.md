WYLTJ-Rust-Compiler 是一个面向编译原理课程实践的 Rust 子集编译器演示项目。项目基于 Vite + React + TypeScript 构建，将源代码编辑、词法分析、语法分析、语义检查、中间代码生成和结果可视化整合在同一个交互界面中，适合用于课程展示、实验验证和编译流程学习。

项目关注的不是完整复刻 Rust 语言，而是围绕教学场景实现一条清晰、可观察、可扩展的编译流水线：输入类 Rust 代码后，系统会依次生成 Token 序列、AST 语法树、语义诊断信息、中间代码和目标代码展示结果。

## ✨ 功能模块

### 🔤 词法分析

**1. WHAT**

词法分析模块负责把用户输入的源代码拆分成编译器可以处理的 Token 序列。当前支持关键字、标识符、整数、浮点数、运算符、比较符、分隔符、括号、数组下标、引用符号、函数返回箭头和区间表达式等词法单元。

**2. HOW**

项目在 `src/app/compiler/lexer.ts` 中实现手写扫描器，按字符顺序读取源码，并根据当前字符类别进入不同的扫描逻辑。标识符和关键字通过关键字表区分，数字字面量支持整数和浮点数识别，多字符符号如 `->`、`==`、`!=`、`<=`、`>=`、`..` 会在扫描阶段合并为独立 Token。每个 Token 同时保留行列信息，用于后续错误定位。


### 🌳 语法分析

**1. WHAT**

语法分析模块将 Token 序列构造成抽象语法树 AST。项目支持函数声明、参数和返回类型、变量声明、赋值语句、返回语句、表达式语句、代码块、`if / else if / else`、`while`、`for`、`loop`、`break`、`continue`、函数调用、数组、元组、引用、解引用、下标访问和成员访问等结构。

**2. HOW**

项目在 `src/app/compiler/parser.ts` 中实现递归下降解析器，并使用 Pratt Parsing 处理表达式优先级。语句级结构由专门的解析函数分派处理，表达式级结构通过优先级表统一解析，从而正确处理 `a + b * c`、函数调用、数组下标、成员访问和区间表达式等组合形式。解析得到的 AST 节点定义在 `src/app/compiler/ast.ts` 中，结构清晰，便于语义分析和中间代码生成复用。


### ⚙️ 中间代码生成

**1. WHAT**

中间代码生成模块将语义通过后的 AST 转换为四元式风格的中间表示。它覆盖函数入口、参数、变量声明、赋值、算术运算、比较运算、函数调用、返回、条件跳转、循环跳转、数组访问、元组访问、引用和解引用等编译场景。

**2. HOW**

项目在 `src/app/compiler/ir.ts` 中遍历 AST，并为表达式生成临时变量，为控制流生成标签和跳转指令。`if` 语句会被转换为条件跳转和分支标签，`while` / `for` / `loop` 会被转换为循环入口、条件判断、循环体和回跳结构。表达式求值结果统一落到临时变量或目标位置中，最终以便于阅读的四元式列表展示。


### 🖥️ 可视化交互界面

**1. WHAT**

可视化交互界面提供源码编辑、文件加载、文件保存、一键编译、控制台日志、AST 查看、Token 查看、中间代码查看和目标代码查看等功能。用户可以在同一页面完成从输入源码到观察编译结果的完整流程。

**2. HOW**

前端使用 React 组织页面结构，核心入口在 `src/app/App.tsx`。用户点击编译后，界面调用 `src/app/compiler/analyzer.ts` 中的 `analyzeSource`，该函数串联词法分析、语法分析、语义分析和中间代码生成，并把不同阶段的结果返回给界面组件。各类结果通过编辑器、控制台和标签页分区展示，保证编译流程清晰可追踪。



## 🧠 语义分析能力

除上述展示模块外，项目还内置语义分析阶段，用于保证中间代码生成前的程序具有基本语义正确性。当前支持变量声明和作用域检查、类型一致性检查、函数调用参数检查、返回值检查、可变性检查、未声明变量检查、未赋值变量检查、数组和元组访问检查，以及 Rust 风格引用和可变引用的部分借用规则检查。

## 🚀 快速开始

```bash
npm install
npm run dev
```

生产构建：

```bash
npm run build
```

构建产物会输出到 `dist/` 目录。
